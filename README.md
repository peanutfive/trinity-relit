# Trinity Relit

让经典 Infocom 文字冒险游戏重新可玩。

---

## 为什么是 Trinity

`Trinity (1986)` 是 Infocom 最受推崇的作品之一。它的结构很特别：游戏从 WWIII 爆发、伦敦被核弹摧毁的瞬间开始，你逃入另一个时空，穿越一系列诡异的维度和谜题，最终抵达 1945 年新墨西哥州的沙漠——人类第一颗原子弹引爆前的几分钟。游戏的名字，正是那次核试验的代号：Trinity。

起点是核战争的后果，终点是核武器的诞生。这条线索设计得很克制，也很难忘。

今天能玩到它的人很少了——需要安装解释器、在终端操作、用英文输入指令。门槛不在于游戏本身，在于运行环境和语言。Trinity Relit 从这里出发，试验几种降低门槛的方式。

---

## 方案说明

### 方案 A：保留原作，加一层中文接口

`trinity_cn.py` 不动原版游戏逻辑，只在外面包一层 Gemini API：游戏输出翻译成中文，玩家的中文输入翻译成游戏命令。

```bash
# 需要: dfrotz, python3, google-genai, Gemini API Key
python3 trinity_cn.py
```

- 游戏输出实时翻译成简体中文
- 支持中文指令输入（"往北走"、"拿起伞"）
- 输入 `/原文` 显示上次英文原文

### 方案 B：浏览器端语义匹配原型

`prototype/` 完全在浏览器本地运行，不需要服务器，不需要 API Key。

核心思路：文字冒险的事件空间是有限的，玩家输入不需要生成式回答，只需要找到语义最接近的已知事件并执行。用 `multilingual-e5-small` 嵌入模型（~130MB）计算余弦相似度来做匹配，离线可用。

目前包含 Trinity 开场的 6 个场景，有一条完整可走通的路线。

```bash
cd prototype
python3 -m http.server 8080
# 浏览器打开 http://localhost:8080
```

- 无服务器、无 API Key，完全离线可用
- 中文/英文指令均支持
- 内置调试面板，可查看语义匹配分数

---

## 游戏文件

出于版权原因，仓库不包含原版游戏文件。运行方案 A 需要自行准备 `TRINITY.DAT` 并放到对应目录。
